## Overview
This repository is a **QA Automation portfolio project**. The mini e‑commerce application serves solely as a realistic backdrop to demonstrate comprehensive testing practices, including API automation, end‑to‑end UI testing, data validation, and reporting. The focus of this README is to highlight the testing strategies, tools, and competencies implemented throughout the project.

## Project Objectives
- Automate functional testing of the REST API and the user interface.
- Achieve meaningful test coverage across critical user flows (authentication, CRUD operations, checkout).
- Produce structured HTML reports and evidence artifacts for each test run.
- Apply industry‑standard best practices such as the Page Object Model, fixture management, and CI‑ready test pipelines.

## Features (QA Capabilities)
- **API Test Automation** – comprehensive Jest/Supertest suites covering success, error, and edge‑case scenarios.
- **End‑to‑End Testing** – Playwright scripts exercising the full UI, including smoke and regression suites.
- **CRUD Validation** – tests verify creation, retrieval, update, and deletion of resources.
- **Authentication Testing** – JWT‑based login flows validated both at API and UI levels.
- **Smoke & Regression Suites** – tagged tests (`@smoke`, `@regression`) for quick feedback and full regression runs.
- **Page Object Model** – reusable UI components encapsulated in Playwright page objects.
- **Test Documentation** – structured test cases and plans (to be added under `docs/`).
- **HTML Reports** – generated by `jest-html-reporter` and Playwright report server.
- **Evidence Collection** – screenshots and logs stored in `docs/evidence/` after each run.

## Testing Strategy
The testing effort targets the most critical user journeys:
- **Automated API tests** cover all public endpoints, including authentication, CRUD operations, and checkout flow.
- **Playwright E2E suites** validate the UI representation of those same journeys, exercising both happy‑path and negative scenarios.
- **Smoke tests** (`@smoke`) provide a quick sanity check on CI.
- **Regression tests** (`@regression`) run on every pull request to guarantee no breakage.
Features not automated include performance benchmarking and accessibility testing, which are planned for future iterations.

## Stack of Testing
- **Jest** – unit and integration tests for backend logic.
- **Supertest** – HTTP assertions against the Express API.
- **Playwright** – end‑to‑end browser automation.
- **jest‑html‑reporter** – HTML test reports.

## Installation
```bash
# Clone the repository
git clone https://github.com/gianluqqa/booker-mini-eccomerce.git
cd booker-mini-ecommerce

# Install dependencies for both back‑ and front‑end
npm run install:all
```

## Environment Variables
Copy `.env.example` to `.env` and adjust values as needed:
- `DB_HOST`
- `DB_PORT`
- `DB_USERNAME`
- `DB_PASSWORD`
- `DB_NAME`
- `ADMIN_EMAIL`
- `ADMIN_PASSWORD`

## Running the Project
```bash
# Development (both services)
npm run dev

# Backend only
npm run dev:backend

# Frontend only
npm run dev:frontend
```

## Running Tests
```bash
# Backend tests
npm run test            # All Jest tests
npm run test:watch      # Watch mode
npm run test:coverage   # Coverage report
npm run test:integration

# Frontend (E2E) tests
npm run test:e2e        # Full Playwright suite
npm run test:smoke      # @smoke tagged tests
npm run test:regression # @regression tagged tests
```
